Setup Guide

Get VS Code, the VEX extension, and the VEXLearn template installed and configured on your machine.

VS Code VEX Extension Windows

Installing VS Code

Visual Studio Code is the recommended code editor for VEXLearn. Follow these steps to install it.

  1. Open your browser (Chrome, Edge, Firefox, etc.)
  2. In the address bar, type code.visualstudio.com and press Enter
  3. Click the Download for Windows button and wait for the file to download
  4. Open the downloaded installer file
  5. Accept the license agreement and click Next
  6. On the "Select Additional Tasks" screen, make sure Add to PATH is checked (the rest is optional)
  7. Click Install and wait for it to complete
ℹ️

You can also use other editors like Antigravity, but VS Code is the most widely tested with the VEX extension.

Installing the VEX VS Code Extension

The VEX Robotics extension adds support for building and downloading VEX programs directly from VS Code.

  1. Open Visual Studio Code
  2. Click the Extensions icon in the left sidebar (or press Ctrl+Shift+X)
  3. In the search bar, type vex
  4. Find VEX Robotics (by VEX Robotics) in the results and click Install
  5. Wait for the extension to finish installing
⚠️

PROS is not supported. VEXLearn only works with the official VEX VS Code extension and V5 platform (not IQ or EXP).

Installing the Template

Prerequisites

  • You have VS Code (or another editor) installed
  • You have the VEX Robotics extension installed
  1. Download the ZIP file from the releases page (link is in the README section of the GitHub repo)
  2. Extract the ZIP file to a location on your computer
  3. Open VS Code, go to File → Open Folder, and open the extracted folder.
    🚨

    The folder you open must be named VEXLearn Template. When you extract the ZIP, a parent folder may be created with VEXLearn Template inside it. Open the inner folder. If you open the outer parent folder, the VEX extension will not recognize the template.

  4. Let VS Code install the required dependencies automatically.
    ⚠️

    If a popup asks "Do you want to install Microsoft's Makefile tools?", click No. Installing it will break the template.

Learning Only vs. For Use

Depending on your goal, the setup is different. Choose your path below.

✅

If you just want to explore how the code works and learn from it, no extra setup is needed beyond installing the template. You're done!

Browse the source files to understand how the template is structured:

  • src/main.cpp: Entry point, thread setup, pre-auton calibration
  • src/drive.cpp: Driver control and button mappings
  • src/movement.cpp: PID-based move and turn functions
  • src/auto.cpp: Autonomous functions
  • src/display.cpp: Brain and controller screen rendering
  • src/robotConfig.cpp: Motor, sensor, and piston objects

Your robot must meet these hardware requirements:

  • Tank-drive chassis (no X-drive, H-drive, or mecanum)
  • All chassis wheels must be the same size
  • Clear split between left and right chassis motor groups
  • Omni wheels and traction wheels are both fine
  • Must have an inertial sensor
  • Only V5 motors (not EXP or IQ)
  • Only one controller

Robot Setup Checklist

Click each step to mark it complete as you go:

0 / 14 done
  • Go to vex.h and change wheel_2r to your wheel diameter
  • In robotConfig.cpp: add/remove motors with correct gear ratio, port, and direction
  • In robotConfig.cpp: add/remove sensors with correct port number
  • In robotConfig.cpp: add/remove pistons with correct three-wire port
  • In robotConfig.cpp: update motor groups so motor names match
  • In robotConfig.h: initialize the same objects created in robotConfig.cpp
  • In movement.cpp: update the gear ratio in the move function (default is 4/3)
  • In drive.cpp: delete examples, write your button mappings
  • In main.cpp: update pre_auton to calibrate your sensors
  • In display.cpp: remove motors that don't exist, update team name/number
  • Compile the project, no errors (warnings are fine)
  • Run on the robot and verify it works. Adjust driver preferences (deadband, sensitivity)
  • In auto.cpp: write a simple PID test function (e.g. move(100, 0, 8);) and tune PID
  • In auto.h: declare your PID tuning function. Robot is ready!
💡

When you're done with setup, start tuning PID for move() and turn() in movement.cpp. See the VEX Concepts → PID section for tuning guidance.
